# Safe Upgrade Plan: OpenClaw 2026.5.2

Date: 2026-05-03

## Summary

Upgrade only after a rehearsal proves the `2026.4.24 + local patches` queue works on
`v2026.5.2`. Do not touch the live `local-patches` gateway until rehearsal passes.
Keep rollback to the current frozen commit ready.

Current baseline:

- Live tree: the canonical `local-patches` worktree from the maintenance skill
- Live version: `v2026.4.24-12-g42aa939db6`
- LaunchAgent commit: `42aa939db611b365db5d9ccd0dbca5eb7f5ad2e7`
- Target tag: `v2026.5.2` at `d07e13380c24b131d6a9260fbb6a28a35e6241e7`

## Key Changes To Preserve

- Discord account-scoped channel routing.
- Active-memory/QMD latency and fallback behavior.
- Pi clipboard guard for headless gateway startup.
- Gateway wrapper fixes for `pnpm openclaw gateway ...`.
- Calendar, heartbeat, and agent tooling compatibility surfaces already restored.

Do not run `npm install -g openclaw@2026.5.2` as the first step. The live gateway
runs from local `dist/index.js`; global CLI changes are last, optional, and only after
the local runtime passes.

## Rehearsal Flow

1. Back up live state.
   - Record current commit `42aa939db6`.
   - Copy the active OpenClaw config and LaunchAgent plist to timestamped backups.
   - Confirm agent workspaces remain committed and no migration is needed there.

2. Prepare an isolated upgrade.
   - Fetch upstream tag `v2026.5.2`.
   - Create a sibling rehearsal worktree, for example `rehearsal-2026.5.2`.
   - Base it on `v2026.5.2`.
   - Replay `v2026.4.24..maint/local-patches`.
   - Resolve conflicts in favor of preserving local runtime behavior, not upstream defaults.

3. Rehearsal verification.
   - `pnpm config:channels:check`
   - `pnpm config:schema:check`
   - `pnpm build`
   - Targeted tests for active-memory timeout/quota behavior, memory-core QMD fallback,
     gateway CLI wrapper behavior, and Discord routing/channel config.
   - Run a non-live temp gateway smoke on an alternate port with live channels disabled.

4. Rehearsal acceptance.
   - No `ERR_MODULE_NOT_FOUND`.
   - No native clipboard load in the headless gateway path.
   - No lazy runtime dependency install during live request handling.
   - Config preserves Discord bindings and agent model settings.
   - Build metadata reports `2026.5.2`.

## Live Cutover

Only after rehearsal passes:

1. Stop the live launchd gateway.
2. Move `local-patches` to the rehearsed upgrade result.
3. Run final checks in `local-patches`:
   - `pnpm config:channels:check`
   - `pnpm config:schema:check`
   - `pnpm build`
4. Refresh the service:
   - `node dist/index.js gateway install --force`
   - `launchctl kickstart -k gui/$(id -u)/ai.openclaw.gateway`
5. Verify:
   - `pnpm openclaw gateway probe`
   - `pnpm openclaw channels status`
   - launchd metadata shows `OPENCLAW_SERVICE_VERSION=2026.5.2`
   - logs show Insa, Laylah, Peggi, and Zach connected
   - logs show no stale chunk import errors, no native clipboard hang, and no runtime staging
     during inbound requests

## Rollback Plan

If live cutover fails:

1. Stop the launchd gateway.
2. Return `local-patches` to `42aa939db6`.
3. Run `pnpm build`.
4. Reinstall and restart launchd.
5. Verify gateway probe and channel status.
6. Do not push the failed upgrade branch.

Rollback is successful only when launchd metadata is back to `2026.4.24` and
`pnpm openclaw channels status` is healthy.

## Assumptions

- Upgrade target is the release tag `v2026.5.2`, not upstream `main`.
- No global npm/pnpm CLI upgrade until the local source runtime is proven.
- A short live downtime window is acceptable only for final cutover.
- Agent workspaces stay unchanged unless the new runtime requires an explicit compatibility fix.
